物理层

物理层没有什么协议，需要掌握的内容是

  • 频分、时分、统计时分、波分和码分技术的原理
  • 信噪比，香农公式的了解
  • 常见传输媒介的优劣

频分复用

多个用户在同一时间占用不同的频带

时分

多个用户在不同的时间占用同一个频带

统计时分

为了解决某些用户没有数据还占用时间片的情况，使用一个集中器讲不发送数据的时间片给别人

波分

这就是光的频分复用技术

码分

多个用户在同一个时间使用同一个频道，发送不同的码型
需要保证，各个站点选取的码片是正交的

信噪比

信号的平均功率和噪声的平均功率之比： S/N
香农公式： C = W * log2(1 + s/n)
C 是极限传输速度，W 是信带宽

CSMA/CD协议

碰撞检测协议，设端到端传输时延为 T，那么争用期是 2T，在争用期期间，有可能发生碰撞，如果争用期没有碰撞，就不会再产生碰撞。如果产生碰撞，使用随机数二进制退让算法进行规避，如果在多次规避之后还是有碰撞，说明不适合发送，返回错误。

TCP 协议

  1. TCP 协议格式
    TCP 连接的端点是套接字

    socket = ip : port

  1. TCP 协议特点
  • 面向连接的运输层协议
  • 可靠交付
  • 全双工通信
  • 面向字节流：多个应用程序包，可能被打包为一个
  1. 可靠传输的保证
  • 低效率传输协议： 停止等待协议：每发送完一个分组就停止发送，等待对方的确认，收到确认之后再继续发送。如果出现差错进行超时重传。对于重复收到的包，接收方采取俩个行动：1.丢弃重复的包。2.重新发送这个包的确认。
  • 高效率传输：连续 ARQ 协议和滑动窗口协议
  • ARQ 协议：对一个窗口的包进行发送和确认，采用累计确认的方式，对最后一个分组发送确认。缺点是可能会因为前面的分组没有收到导致后面的分组进行重传。
  • 滑动窗口协议：三个指针，发送窗口指针、发送未确认指针、允许发送但没有使用指针

​ 只要 TCP 连接的一方收到对方的零窗口通知，就启动持续计时器。计时器设置的时间到了，就发送一个零窗口探测报文段，以防止另一方有空余窗口而本方不知道和传输包丢失的情况。

  1. 超时重传时间
  2. 拥塞控制方法
  • 慢开始：每轮次将发送报文段变为上一次的 2 倍
  • 拥塞避免：当当前窗口大于门限值的时候，每个轮次增加一个报文段
  • 快重传：收到三个重复确认
  • 快恢复：快重传之后进行拥塞避免
  • 门限值调整时机
    • 发生超时，ssthrest = cwnd / 2, 同时拥塞窗口设置为 1
    • 快恢复：ssthresh = cwnd / 2

    四挥手的过程为什么等待 2MSL
  • 为了保证 A 的最后一个报文段能够到达 B。这个报文如果丢失，我们就可以收到 B 的FIN + ACK 重传报文
  • 防止已经失效的请求报文段出现在本连接中。2MSL 的等待时间可以让本连接发送的报文在网络上消失。

保活计时器：为了防止客户端挂掉。在收到客户端的数据之后就会重置保活计时器，如果俩个小时没有收到客户端的数据，就会每隔 75s 发送探测报文，如果 10 个探测报文都没有回复，服务器主动关闭连接。

UDP 协议

  1. UDP 的格式
  2. UDP 的特点
  • 无连接
  • 尽最大努力交付
  • 面向报文：应用程序交付的报文直接进行包装，不会进行切片操作。
  • 没有拥塞控制
  • 支持一对一、一对多、多对多交互通信
  1. 注意点
    UDP 的校验和是把首部贺数据部分放在一起进行检验。
