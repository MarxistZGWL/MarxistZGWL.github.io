putVal逻辑

  1. 如果表还是null或者长度为0, 那么首先进行扩容
  2. 如果哈希桶是null, 那么直接将当前元素放在哈希桶中
  3. 判断哈希桶剩余的元素是否有和当前元素相等的
    • 如果有, 那么进行更新操作, 返回旧值
    • 如果没有, 那么直接插入链表末尾或者树中, 同时更新modCount和size属性

可以看到，三种构造函数最终调用的都是同一个，且在构造函数中不进行哈希桶的初始化流程。

哈希桶的初始化流程，直到在第一次插入元素的时候，才会开始。
